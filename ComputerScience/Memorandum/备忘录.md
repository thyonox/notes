# 远程 SSH
我需要登录远端主机

一般可以通过密码，方便点的可以使用ssh

现在本机生成ssh的公钥和私钥，将私钥本村本机，公钥传给远端主机

生成ssh密钥对

```jsx
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

- `t rsa` 指定密钥类型为 RSA。
- `b 4096` 设置密钥的位数为 4096 位（更安全）。
- `C` 添加注释（通常为你的电子邮件地址）。

默认情况下，它会保存到 `~/.ssh/id_rsa`

生成密钥后，你需要将公钥内容添加到远程主机的 `~/.ssh/authorized_keys` 文件中。

将公钥复制到远端主机

```jsx
ssh-copy-id username@remote_host
```

如果ssh-copy-id不可用

```jsx
ssh username@remote_host
mkdir -p ~/.ssh
echo "your-public-key-content" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

确保远程主机的 `~/.ssh` 目录权限设置正确，且 `authorized_keys` 文件的权限为 `600`（仅所有者可读写）。

# m3u8视频下载
### M3U8和TS

m3u8是一种播放列表文件格式，而不是视频容器或者编码格式。

m3u8文件使用UTF-8编码，主要作用与流媒体协议中，如HSL（HTTP Live Streaming），包含了一个或多个媒体片段（如TS文件）的信息，播放器可以通过这些信息顺序地加载并播放媒体片段，这样做的好处是可以使视频更快的播放，在跳转观看大文件视频时更加丝滑，也可以根据网络状况灵活的切换视频码率，而且在某个媒体片段丢失损坏时只需要重新下载对应的视频片段即可，并且因为m3u8和HLS支持多种加密格式，可以为视频创作者和视频平台提供更好的版权保护。

TS如MP4、GIF、ICO等是一种容器格式，用于多路复用视频流、音频流和数据流，因为可以有效地处理错误和数据丢失，因此常用于不稳定的网络环境，现在广泛应用在流媒体中。

# 认知负荷理论
认知负荷理论（Cognitive Load Theory，CLT）由John Sweller于20世纪80年代发表的论文《Cognitive load during problem solving: Effects on learning》中提出，用于解释人类在学习和问题解决过程中所经历的认知负荷，探讨了在问题解决过程中，认知负荷对学习效果的影响。

认知负荷过重（Cognitive Overload）是指当一个人的工作记忆容量被过量的信息或任务占据，超出了他们能够有效处理的范围时所产生的一种状态。简单来说，就是当大脑一次性接收或处理的信息量过大，超过了它的处理能力，导致理解、学习和决策变得困难。

该理论主要基于工作记忆与长期记忆。工作记忆是处理和存储信息的临时系统，容量有限，一般认为只能同时处理5到9个信息单元。长期记忆是储存信息的永久系统，容量几乎是无限的。通过有效的学习和练习，信息可以从工作记忆转移到长期记忆。

Sweller通过一系列实验研究了不同教学方法对学习者认知负荷和学习效果的影响。实验主要集中在数学和科学领域，比较了传统问题解决方法与基于示例学习（worked examples）的效果。实验结果显示，基于示例的学习方法比传统的独立问题解决方法更有效。学习者通过示例学习可以减少不必要的认知负荷，更好地理解和记住解决问题的步骤和策略。传统的独立问题解决方法会导致较高的认知负荷，因而影响学习效果。反之，示例学习方法能够通过提供明确的步骤和指导，减少学习者的认知负荷。论文强调了有效的教学设计应该关注减少学习者的认知负荷，建议通过分步引导和提供详细示例来优化教学。

Sweller在论文中提出了认知负荷的三种类型：内在负荷（Intrinsic Load）、外在负荷（Extraneous Load）和本质负荷（Germane Load）。内在负荷与任务本身的复杂性有关，无法通过教学设计改变，例如，学习微积分比学习简单的加减法具有更高的内在负荷。外在负荷是由于不良的教学设计或多余的信息引起的，例如，不断刷短视频，因为短视频通常信息量大且内容变化快，大脑需要不断地处理新信息，并且这些信息通常是不相关的，导致认知系统被频繁打断和重启。本质负荷与学习过程中的深度处理和理解有关，这是对工作记忆产生积极影响的负荷，促进知识的建构和转化。例如，在《如何高效学习》一书中所说将单独知识汇入整体认知中，从不同角度理解新知识，铺设出一条大脑的高速公路。针对不同的认知负荷，Sweller强调了通过优化内在负荷、减少外在负荷、增加本质负荷的方式设计教学和学习策略。

Sweller通过这些概念的提出，为教育心理学和教学设计提供了一个有力的框架。在如今知识爆炸以及人工智能在越来越多行业替代传统人工的当下，一个人从一无所知到进入某一行业所要求的知识将会越来越多，如何高效学习，缩短学习时间，不但是教育者应该考虑的问题，也是每一个人应该具备的能力。

# USB接口类型及协议
> 日常生活中，USB接口是人们常用常见的，但人们对USB接口没有过多了解，称呼不一而足，所以在这里总结一下。文档仅包含USB概述、USB接口类型以及USB协议，以作科普。

### 概述

USB（通用串行总线，Universal Serial Bus）是一种连接计算机及其外部设备的标准接口，主要应用于数据传输、供电等方面。

在计算机发展早期，外接设备的接口各不相同，也没有考虑到热插拔的需求，繁杂的接口系统加上必须安装驱动并重启才能使用的限制，造成了用户使用困难，因此，创造出一个统一且支持易插拔的外界接口，就成了无可避免的趋势，于是USB应用而生。

1994年，由英特尔和微软等七家公司共同发起了USB标准的开发，目的是创建一种统一的接口标准，尽可能实现热插拔，简化计算机和外部设备的连接。

USB标准发展到今天，经过了USB1.x、USB2.0和USB3.x，到如今的USB4，衍生了USB OTG（On-The-Go）拓展标准，允许支持该标准的设备在没有PC的情况下直接互相通信。

### 类型

USB接口类型主要分为USB Type-A、USB Type-B和USB Type-C，其中，USB Type-A和USB Type-B又分为Standard（标准）、Mini（小型）和Micro（微型）三种类型。

USB Type-A最常见的是Standard Type-A USB接口，外形为矩形接口，支持USB3.x并且向下兼容，其余类型以及一些变体比较少见，主要适用于USB2.0，逐渐被Type-B的Mini和Micro类型以及Type-C取代。Standard Type-A USB又分为支持USB1.x和USB2.0的四引脚版本和支持USB3.x的九引脚版本。

USB Type-B最常见的是Micro Type-B USB接口，因为常用于老式安卓机，也被称为“安卓口”，5个引脚，仅支持USB2.0，至于对USB3.x的支持则是在旁边拓展了5个引脚。Standard Type-B USB接口分为支持USB2.0的4引脚版本和支持USB3.x的9引脚版本。Mini Type-B USB接口仅有支持USB2.0的5引脚版本，主要用在一些老式MP3等便携设备上。

USB Type-C只有一种形态，双面可插，USB4唯一支持的USB接口类型，向下兼容USB2.0，苹果笔记本第一个搭载了Type-C接口，第一个搭载Type-C接口的手机为乐视超级手机1，但不知为什么常被称为“华为口”，C口或者Type-C相比更合适一些。

# 通货膨胀
通货膨胀简单来说就是钱的贬值，最常见的衡量指标叫做 CPI（Consumer Price Index 消费者物价指数），代表商品和服务的物价走势。

[区域欺骗](https://www.yuque.com/suwuji/kr13ng/pz5titdetlgkwlt1)

下图是美国近60年的 CPI 走势图：

![](https://cdn.nlark.com/yuque/0/2023/png/29046015/1676374034611-db0158cc-dd67-4f24-9fa0-53df99f5e8df.png)

可以看到60年前是29，现在是296，绝对的数字意义并不大，看对比就知道：现在的物价相当于60年前的10倍。也就是说60年前的1美元相当于现在的10美元。

物价上涨，代表钱的贬值，也就是说发生了通货膨胀。

经济学家用来形容和解释通货膨胀的公式有很多，比如：凯恩斯主义、货币主义、新凯恩斯主义等等。但是这些公式并不能一以贯之的完全解释通货膨胀。可能在我们看来，通胀的形成原因很简单，无非经济体中的货币量超过了经济的体量，太多的货币去追逐太少的商品，从而造成物价的上涨。

各国央行的首要目标或者很多央行的唯一目标就是维持物价的稳定，钱随便印、利率随便调，只要保持物价就可以，但就是这么简单的事，就是拿捏不住，而且像美联储、欧洲央行里面汇聚全球顶级经济学家，他们内部意见也并不同意，可见问题并没有这么简单！

如果我们站在整个经济体的角度，那钱无非就是一个流通的工具。所以这个货币对应的购买能力，比如一杯奶茶3块钱还是300块钱，其实并不重要。我们追求的是更快乐、更幸福，有更多的商品可供选择。对应经济体中的更高的产值，也就是更高的GDP，这样整个经济机器的运转也就会更高效。而钱在经济体中扮演商品交换的媒介，所以我们肯定是希望钱的购买能力能够更稳定。

但是就有人发现，所有人手里的钱都是在一点点贬值的，就算你将一箱子钱藏在床底下，钱并没有变得更少，但是钱的购买能力是在一点点下降的。而这些消失的购买力就被转移到新创造出来的钱上，在这一持续不断的微调影响下，就逼得个人或团体去消费、去生产、去挣更多的钱，也就是说通货膨胀其实是一种无时无刻不在进行的财富的再分配。

一滩水中添加了通货膨胀后，就会迸发出更大的活力，这也就是为什么有人说适量的通胀对整个经济体来说是更好的。

照着通胀的逻辑，如果相同的钱明年购买的东西比今年还多，也就是出现通货紧缩会发生什么？

那大家肯定是不花钱或者尽量少花钱，毕竟只要不花钱就相当于在赚钱，这样就会导致经济体中总需求下降，长期来看会使经济体进入非常可怕的通缩恶性循环。

最典型的例子是日本90年代泡沫破裂之后，就陷入了通货紧缩，之后就是信用萎靡，经过了30年，央行不管怎么刺激、政府不管怎么借贷，需求就是刺激不起来，抛出老龄化等问题，也可以看出来通货紧缩是一个多么可怕的问题。

对于通货紧缩，价格越来越便宜并不是真正的问题，而是大家都不工作、不努力、不挣钱、不生产、不消费，进入了低欲望社会，这才是深层的问题。

我们关心通货膨胀不是因为它会带来价格的变化，而是价格的变化会带来需求的变化和GDP的变化。

但是通货膨胀这个财富再分配的响应也不能太过了，比如我今天手里的钱能买100杯奶茶，明天就只能买50杯了，后天可能一杯都买不到了，大家只要是有钱在手，肯定立马花掉，这种情况就叫超高速的通货膨胀也就是恶性通胀（HyperInflation）。它对经济的打击是毁灭性的，金融系统、信贷系统都将完全瘫痪，社会生产力骤减，进入恶性通胀的国家几乎无一例外要经过艰苦卓绝的全面改革、全面重启才有可能慢慢恢复过来。

上面介绍的通货紧缩就当于经济给冻住了，不发展了，而恶性通货膨胀相当于经济过热了，开始自燃了。

这也就是为什么大多数的经济学家都会追求那个适量的通货膨胀，其实就是一个度的问题。

![](https://cdn.nlark.com/yuque/0/2023/png/29046015/1676378146256-d8c42d60-7c1a-4301-9fd6-e067a19808fc.png)

那通货膨胀对于资产价格，比如股价和房价有什么影响呢？

这里面干扰因素有很多，如果直观去猜的话，可能会觉得通货膨胀和资产价格是正相关，因为大家为了抵御通胀会将资产投入房地产或股票市场中。但其实这两者的关系是负相关，下面是IMF对71个经济体过去股价和通胀之间关系的回归分析：

![](https://cdn.nlark.com/yuque/0/2023/png/29046015/1676379123746-d0559b4f-095d-4d5d-9e2a-99c86272183f.png)

其结论就是不管是发达国家还是发展中国家，股价的表现和通货膨胀是负相关关系。这其中主要原因是政策才是促使股票下跌的原因，在高通胀时期，政府一般都会制订紧缩政策，而这紧缩政策就导致了股价的下跌。

这就是为什么土耳其是一个例外，因为在高通胀时期并没有采取紧缩政策，使得股价一年之内翻了3倍，还有一个例外就是能源公司，历史数据表明，在高通胀时期，能源公司表现都非常好。

导致通胀的大概有三种因素：

第一：需求拉动型的通胀（Demand Pull）。

比如在一个真正运转的经济体中，奶茶老板疯狂打广告、做营销，搞得大家都非常想喝奶茶，大家就会跑去奶茶店买奶茶，这时奶茶的需求上升，就会导致两个结果：奶茶产量上升、奶茶涨价。不管怎样奶茶老板赚到了钱，良心老板还会给员工发奖金，老板和员工有钱了买包买吃的，就会导致食品和包的产量上升以及价格的提高。

![](https://cdn.nlark.com/yuque/0/2023/png/29046015/1676379784360-5aa02439-86fa-49f2-9b23-c821714c29ba.png)

总结来说：需求增加，就会导致产量增加和价格上涨，使企业利润增加，就会导致消费增加，就又进入了新一轮的需求增长。

![](https://cdn.nlark.com/yuque/0/2023/png/29046015/1676380199575-89684e24-760f-44bc-902a-c7aefec45235.png)

但需求增长的同时就会导致价格的提高，也就是通货膨胀。

但经济也确实提高了，所以政府很多情况下想刺激经济就会从总需求下手，只不过副作用是通货膨胀。

什么是刺激需求？说白了就是刺激大家花钱消费，所以说要么政府自己花钱，比如基建、项目补贴等，要么就是直接将钱发到大家的手里，买什么自己决定。

所以就说通货膨胀和经济增长就像一个绑定的套装，它是买一送一的。虽然说它是买一送一的，但只要刺激的时间点合适，其实也可以做到经济增长的同时，尽量少的带来通货膨胀。

比如说现在政府花了一万亿刺激大家去买奶茶，假设这个奶茶店就是人都闲着，材料也非常充足，这样新顾客来了我直接做就行了，非常容易满足需求，那奶茶几乎就不怎么涨价，而只增长了GDP，反过来假设这个奶茶店本身就已经人满为患了，这时候政府要是非要还去刺激需求，那就只能造成奶茶涨价，而产量本身并没有增加。

他们最大的区别在于这个奶茶店还有没有一些没被释放出来的产能，假设这个经济当中蕴藏着非常巨大的产能的时候，钱砸进去看到的就是经济的增长。

那我们怎么才能看这个经济当中产能情况呢？

其实你可以去看看失业率，如果失业率很高，那就说明有很多人没在工作，就有很多产能。反过来假设失业率很低，那就说明大部分的劳动力都去工作了，这种情况下产能就很吃紧，这也说明了为什么高通胀之前，都会伴随着一个失业率很低的阶段，其实凯恩斯主义里面有一个就做菲利普斯曲线，他就是说通货膨胀和失业率之间呈负相关的关系。

大多数国家地区都会把通胀控制在2%-5%之间，各国央行不管用什么政策，头号目标就是把通胀控制在这一合理区间，即使成本是牺牲掉一些经济的短期增长甚至是陷入短期的衰退，保住通胀就是个大发达经济体的一个底线。

# 注册表
### 1.注册表是什么

注册表是在Windows操作系统中使用的一个重要的系统数据库，它用于存储各种系统和应用程序的设置、配置和选项。它包含了操作系统的所有设置和控制，包括硬件、软件和用户信息。它通常被认为是操作系统的大脑，因为所有的配置信息都存储在这里，并且它会在计算机启动时被加载到内存中。

注册表是一个层次结构，由多个键和值组成。键表示一个文件夹或子项，而值则存储键的属性和数据。在注册表中，每个键都有一个唯一的名称和路径，类似于文件系统中的文件夹路径。键和值的层次结构和名称都是由应用程序和操作系统决定的。

由于注册表包含了操作系统的大量设置和配置信息，因此对其进行错误的修改可能会导致系统不稳定甚至无法正常工作。因此，对于不熟悉注册表的用户来说，修改注册表应该非常小心，并且最好备份注册表以防止意外的问题。

### 2.打开注册表

按住 win + R 打开运行框，输入 regedit 即可开打注册表。

### 3.注册表结构

![](https://cdn.nlark.com/yuque/0/2023/png/29046015/1681085585112-9cda605e-d56e-47e5-a13d-a4d00244871c.png)

在Windows注册表中，有5个主要的HKEY（即根键）代表不同的注册表分支，它们分别是：

1. `HKEY_CLASSES_ROOT`（HKCR）：这个键存储了系统中所有已注册的文件类型及其对应的应用程序。例如，如果你双击一个扩展名为“.txt”的文件，系统会通过HKEY_CLASSES_ROOT键查找与之关联的应用程序。
2. `HKEY_CURRENT_USER`（HKCU）：这个键存储了当前用户的个性化配置信息，例如桌面背景、屏幕分辨率、字体等。
3. `HKEY_LOCAL_MACHINE`（HKLM）：这个键存储了计算机的硬件和系统信息，例如安装的设备驱动程序、服务、安装的软件等。
4. `HKEY_USERS`（HKU）：这个键存储了所有用户的配置信息，包括当前用户、系统用户和服务账户等。
5. `HKEY_CURRENT_CONFIG`（HKCC）：这个键存储了当前系统配置信息，例如当前使用的硬件配置信息、驱动程序配置信息等。

提示词与反向提示词规则

免费获取提示词网站：

采样方式

- Euler a：一般情况用这个就可以了
- DPM++ SDE Karras：擅长真实图片，如模拟照片及3D算图
- DDIM：厚涂效果，适用于韩系画风

采样迭代步数：给AI更多的时间去贴近提示词，数值越高当然运算时间更长，而且也不一定能拿到好的结果，数值越小肯定会影响图片质量，一般在20-50之间。

宽高比：设置图片宽高以及分辨率。

随机种子：生成图片后，下方会显示图片的 seed，将值填入，以后生成的图片将会在该图的基础之上进行迭代。
